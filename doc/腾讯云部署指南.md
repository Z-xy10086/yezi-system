# 农作物叶片病斑检测系统 - 腾讯云部署指南

## 一、部署方案对比

| 方案 | 适用场景 | 优点 | 缺点 |
|------|----------|------|------|
| **云服务器 CVM** | 生产环境，需要稳定运行 | 配置灵活，性能可控，适合长期运行 | 成本较高，需要运维 |
| **云函数 SCF** | 轻量应用，流量不大 | 按需付费，成本低，无需运维 | 有执行时间限制，不适合长时间任务 |
| **容器服务 TKE** | 大规模部署，需要弹性伸缩 | 高可用，自动化运维 | 配置复杂，成本较高 |

**推荐方案**：对于本项目（Flask Web应用 + 图像处理），推荐使用 **云服务器 CVM**，因为图像处理可能需要较长时间，云函数有执行时间限制。

## 二、准备工作

### 1. 注册腾讯云账号
- 访问 [腾讯云官网](https://cloud.tencent.com/)
- 注册账号并完成实名认证

### 2. 购买云服务器（CVM）

**推荐配置**：
- **地域**：选择离你最近的地域（如广州、上海）
- **操作系统**：Ubuntu 20.04 LTS（推荐）或 CentOS 7
- **配置**：2核4G（最低配置），带宽1-2M

### 3. 购买域名（可选）
- 在腾讯云购买域名（如 `yourdomain.com`）
- 完成域名备案（国内服务器必须）

### 4. SSL证书（可选）
- 申请免费SSL证书（腾讯云提供）

## 三、部署步骤（云服务器 CVM）

```bash

```

关键配置项：

- `UPLOAD_FOLDER`：上传文件目录
- `MAX_CONTENT_LENGTH`：最大上传大小

### 步骤6：初始化数据库

```bash
# 激活虚拟环境
source venv/bin/activate

# 运行初始化命令
python run.py init-db
```

### 步骤7：使用Gunicorn启动服务

```bash
# 安装Gunicorn（如果未安装）
pip install gunicorn

# 启动服务
gunicorn --workers=4 --bind=0.0.0.0:5000 run:app
```

参数说明：
- `--workers=4`：4个工作进程（推荐设置为CPU核心数×2）
- `--bind=0.0.0.0:5000`：绑定到5000端口

### 步骤8：配置Nginx反向代理（推荐）

```bash
# 安装Nginx
sudo apt install nginx -y

# 创建Nginx配置文件
sudo vim /etc/nginx/sites-available/car_project
```

配置内容：

```nginx
server {
    listen 80;
    server_name yourdomain.com;  # 替换为你的域名或服务器IP

    location / {
        proxy_pass http://127.0.0.1:5000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    location /static/ {
        alias /root/car_project/app/static/;
        expires 30d;
    }

    location /uploads/ {
        alias /root/car_project/uploads/;
        expires 30d;
    }
}
```

启用配置：

```bash
sudo ln -s /etc/nginx/sites-available/car_project /etc/nginx/sites-enabled/
sudo nginx -t  # 检查配置是否正确
sudo systemctl restart nginx
```

### 步骤9：配置防火墙

```bash
# 开放80端口
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
sudo ufw allow 5000/tcp
sudo ufw enable
```

### 步骤10：配置systemd服务（开机自启）

```bash
sudo vim /etc/systemd/system/car_project.service
```

配置内容：

```ini
[Unit]
Description=Flask Application - Car Project
After=network.target

[Service]
User=root
WorkingDirectory=/root/car_project
Environment="PATH=/root/car_project/venv/bin"
ExecStart=/root/car_project/venv/bin/gunicorn --workers=4 --bind=0.0.0.0:5000 run:app
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

启用服务：

```bash
sudo systemctl daemon-reload
sudo systemctl start car_project
sudo systemctl enable car_project

# 查看状态
sudo systemctl status car_project
```

## 四、部署步骤（云函数 SCF）

### 步骤1：准备代码

创建 `scf_entry.py` 入口文件：

```python
import os
import sys

sys.path.insert(0, os.path.dirname(os.path.abspath(__file__)))

from app import create_app

app = create_app()

def main_handler(event, context):
    from werkzeug.wrappers import Request
    from werkzeug.wrappers import Response
    
    path = event['path']
    method = event['httpMethod']
    headers = event['headers']
    body = event.get('body', '')
    
    environ = {
        'REQUEST_METHOD': method,
        'SCRIPT_NAME': '',
        'PATH_INFO': path,
        'QUERY_STRING': event.get('queryString', ''),
        'SERVER_NAME': 'localhost',
        'SERVER_PORT': '5000',
        'SERVER_PROTOCOL': 'HTTP/1.1',
        'wsgi.version': (1, 0),
        'wsgi.url_scheme': 'http',
        'wsgi.input': body,
        'wsgi.errors': sys.stderr,
        'wsgi.multithread': False,
        'wsgi.multiprocess': False,
        'wsgi.run_once': False,
    }
    
    for key, value in headers.items():
        key = key.replace('-', '_').upper()
        if key not in ('CONTENT_TYPE', 'CONTENT_LENGTH'):
            key = 'HTTP_' + key
        environ[key] = value
    
    def start_response(status, response_headers, exc_info=None):
        pass
    
    response = app(environ, start_response)
    return {
        'statusCode': 200,
        'headers': dict(response.headers),
        'body': response.get_data().decode('utf-8'),
    }
```

## 六、常见问题

### Q1：服务器无法访问？
- 检查安全组是否开放端口
- 检查防火墙是否允许访问
- 检查服务是否正常运行

### Q2：上传图片失败？
- 检查上传目录权限
- 检查 `MAX_CONTENT_LENGTH` 设置
- 检查文件大小限制

### Q3：OpenCV 导入错误？
```bash
# 安装缺失的依赖
sudo apt install libgl1-mesa-glx libglib2.0-0 libsm6 libxrender1 libxext6 -y
```

### Q4：如何查看日志？
```bash
# 查看应用日志
sudo journalctl -u car_project -f

# 查看Nginx日志
sudo tail -f /var/log/nginx/error.log
```
